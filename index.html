C++文件IO

发表于 2020-09-27 分类于 C++

在项目中可能频繁使用文件IO，打开文件、读文件、写文件等操作。因此本文对常用的几种方法进行记录。避免使用时混淆。

UNIX 风格

UNIX系统中大多数文件IO只需用到5个函数。这些函数通常被称为不带缓冲的IO，即都会调用内核中的一个系统调用。

open
close
read
write

对于普通文件，写操作从文件的当前偏移量处开始。如果在打开该文件时，指定了O_APPEND选项，则在每次写操作之前，将文件偏移量设置在文件的当前结尾处。在一次成功写之后，该文件偏移量增加实际写的字节数。

lseek

C 标准IO库风格

UNIX风格的IO函数都是围绕文件描述符展开的，成功打开一个文件便返回一个文件描述符，该文件描述符用于后续的文件IO操作。而标准IO库是围绕流展开的，成功打开一个文件便有一个流与之相关联。

对进程预定义了3个流，即标准输入、标准输出、标准错误流，对应于文件描述符的STDIN_FILENO、STDOUT_FILENO、STDERR_FILENO。

标准IO库为了提高效率，提供了带缓冲的IO，这样可以减少read、write系统调用的次数。

3种标准IO缓冲
全缓冲

真正的IO操作是在标准IO缓冲区填满之后执行，可以通过手动调用flush将IO缓冲区内容写入磁盘。

行缓冲

当在输入输出种遇到换行符时，标准IO库执行真正的IO操作。但由于标准IO库来收集每一行的缓冲区长度是固定的，因此只要填满缓冲区，即使未遇到换行符也将IO操作。

不带缓冲

顾名思义不需要缓冲，直接执行IO操作，一般标准错误流stderr是不带缓冲的，为了能将错误即使显示。

更改缓冲类型函数

fopen

type参数指定对该IO流的读写方式。

fread/fwrite

二进制IO，可以读写一个二进制数组，或者读写一个结构体。对于读取文件来说很方便。

fwide

用于设置流的定向，其中流的定向有宽定向和字节定向。宽定向即为一个字符可以用多个字节表示。

注意：fwide函数不改变已定向的流的定向，且没有出错返回。因此在调用前最好清除errno，函数返回时在检查errno。

C++风格IO

C++头文件fstream定义了三个类型来支持文件IO，fstream从一个给定文件读取数据，ofstream像一个给性文件写入数据，以及fstream可以读写给定文件。

open

rdbuf()可以实现一个流对象指向的内容用另一个流对象来输出。

C++风格主要使用fstream得到文件流，并使用rdbuf()将流内容输入到string流中，最后读取(字符)数据。

以上三种方法各有用途。就我个人而言，在Linux下编写网络程序代码会使用第一种方法；从二进制文件读取纯数据会使用第二种方法；从文件中读取内容会使用第三种方法。

浅谈关联式容器一

发表于 2020-09-16

STL关联式容器可分为两大类，第一大类是以RB-tree为底层实现机制。先简要讲述重要的底层数据结构RB-tree。

众所周知关联式容器的存取效率较高，为了提供对数时间的元素插入和访问，底层实现采用了二叉搜索树，更进一步为了避免插入删除操作导致二叉搜索树不平衡情况而导致搜寻效率低下，引入了平衡二叉搜索树。

注：节点的数据结构(本文代码与SGI STL有一定出入，主要是为了理解细节，因此做了些简化)。

AVL-tree

AVL tree要求任何节点的左右子树高度相差最多为1。

插入操作使得平衡被破坏，主要有以下4种方式：

1. 插入点位于X的左子节点的左子树—左左

2.插入点位于X的右子节点的右子树—右右

3.插入点位于X的左子节点的右子树—左右

4.插入点位于X的右子节点的左子树—右左

对于恢复平衡可以采用单旋转和双旋转，这里不具体展开。主要还是介绍容器底层需要用到的红黑树。

RB-tree

红黑树的定义：

1.每个节点不是红色就是黑色

2.根节点为黑色

3.如果节点为红，其子节点必须为黑(父子两节点不得同时为红，但可同时为黑)

4.任一节点至NULL(树尾端)的任何路径，所含之黑节点数必须相同

由以上规则可得出新增节点必须为红色，新增节点的父节点必须为黑，但叶子节点不一定为红，因为调整之后可以为黑。

RB-tree的迭代器

首先RB-tree的迭代器为双向迭代器，但不具备随即定位能力。

STL在设计红黑树时，加了一个实现上的小技巧，即header头节点。header节点为红色，用于区分根节点。且header和root互为对方的父节点，左子节点指向最小节点，右子节点指向最大节点。

至此有了节点结构和迭代器，红黑树的数据结构也可以得到了。

为了与STL标准一致，RB-tree也有begin()和end()函数，由header实现。

对于set、multiset、map、multimap底层都采用红黑树来实现，其主要操作都调用了RB-tree的成员函数。

set

set的特性是所有元素的键值会被自动排序，其实值就是键值，set不允许由两个元素有相同的键值。因为其底层调用了RB-tree的insert_unique函数。

multiset

multiset的特性与set的唯一区别是其允许有重复的元素存在，底层调用了RB-tree的insert_equal函数。
