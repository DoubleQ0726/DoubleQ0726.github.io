给定一个只包含 '(' 和 ')' 的字符串，找出最长的包含有效括号的子串的长度。

示例

题解

为了求解最长的有小括号字串，一般处理最长等问题可以采用动态规划解决。首先初始化一个dp数组，初始值为0。由括号性质可知，当**s[i] = ‘(‘**时，s[i]无法和其之前的元素组成有效的括号对，因此dp[i] = 0；

当s[i] = ‘)’

情况1

s[i - 1] = ‘(‘**，此时状态转移方程为dp[i] = dp[i − 2] + 2**。

情况2

s[i - 1] = ‘)’**，此时如果前面有和s[i]组成有效括号对的字符，就要求s[i - 1]位置必然是有效的括号对(即dp[i - 1] > 0)，否则s[i]无法和前面对字符组成有效括号对。因此如果跳过前面已成对的括号判断且满足s[i - dp[i - 1] - 1] == ‘(‘，则状态转移方程为dp[i] = dp[i − 1] + 2。此时还有一个小问题如果前面已经组成了类似”()”的括号对，则应该把其长度也加上，则状态转移方程变为dp[i] = dp[i − 1] + 2 + dp[i - dp[i - 1] - 2]**。

你的算法时间复杂度必须是 O(log n) 级别。

如果数组中不存在目标值，返回 [-1, -1]。

示例

题解

方法一

因为是在升序数组中查找元素。首先想到的方法即为二分法，因为本题可能有多个相同的元素，因此在用二分法查找到元素之后还需要以找到的元素为中间元素，进行向前向后比较，判断能否找到新的与目标值相同的元素，最后返回下标。

方法二

采用C++STL中lower_bound与upper_bound算法思想，lower_bound返回第一个大于等于目标值的元素的迭代器，upper_bound返回第一个大于目标值的元素的迭代器。两者一组合即可找到数组中目标值的下标。比如数组5,7,7,8,8,10，目标值为8，lower_bound返回第一个8的下标，upper_bound返回第一个大于8的下标(即10的下标)。
